## Слайд 2 — Кейс Барс Груп (~1 мин 15 сек)

«Поговорим о **заказчике**. Продукт — **БАРС-Офис и корпоративные сервисы** Барс Груп. Первая линия поддержки тонет в однотипных обращениях — при этом **ответы уже есть в документации**.

*Пройтись по четырём карточкам.*

**Первая боль — объём.** Сотни страниц инструкций, docx постоянно обновляются. Новый сотрудник не знает, с какого PDF начать.

## Слайд 3 — Архитектура (~1 мин 30 сек)

«На этом слайде — **архитектура целиком**. Это не «ChatGPT с файлами» — это **orchestrated RAG pipeline**.

*Указать на схему ill-arch.svg.*

Пользователь пишет в **Telegram** или **Web**. Внутри **FastAPI orchestrator** — координатор, который прогоняет сообщение через цепочку агентов.

**Qdrant + multilingual-e5** — семантический поиск по проиндексированным docx: мы режем документы на фрагменты, эмбеддинги, картинки из docx тоже попадают в индекс.

**Generator** — LLM формирует ответ **строго по найденному контексту**, не из «головы» модели. Сейчас на демо стоит **GigaChat API** — быстрые ответы для защиты; в **production-контуре** — **Ollama** локально, данные не уходят наружу.

**Evaluator** — confidence gate: если score ниже порога или фрагментов нет — **не галлюцинируем**, а предлагаем **оператора**.

**Escalation** — тикет в очередь, история диалога, панель `/operator`.

**Analytics** — параллельно логирует каждый запрос в Redis: weak spots, тренды — **инструмент для поддержки и техписов**, не для конечного пользователя.

*Легенда внизу.*

Закрытый контур: **Qdrant, Redis, Ollama, Whisper, embeddings** — на сервере. Один вход для пользователя — внутри шесть специализированных модулей.»

**Если спросят «это ChatGPT?»** → «RAG-only: retriever → контекст → LLM → источник. Confidence gate против галлюцинаций.»

**Если спросят про GigaChat** → «На защите — GigaChat для скорости и стабильности demo. Архитектура dual: `LLM_PROVIDER=ollama` для prod, переключается одной переменной в `.env`.»

## Слайд 4 — DocHelper в работе (~1 мин)

«Как это выглядит **глазами сотрудника** — без схем и метрик.

*Показать ill-solution.svg слева.*

Слева — реальный сценарий: человек спрашивает про фильтр или командировку, получает **структурированный ответ**, **фрагмент из docx**, **скачивание документа**, при необходимости — **схему из инструкции**.

*Блок «Было».*

**Было:** листают PDF, пишут в чат, оператор в десятый раз ищет ту же ссылку, **непонятно, из какого документа взята инструкция** — нет audit trail для L1.

*Блок «С DocHelper».*

**Стало:**
- спрашивает в **TG или Web** — как коллегу, можно **голосом**;
- получает ответ **из docx** + кнопку **«скачать источник»**;
- если тема не покрыта docs или бот не уверен — **оператор** получает **полную историю**, не «привет, у меня не работает фильтр» с нуля.

Ключевое: **один вопрос — ответ из корпоративного docx, не из фантазии модели.**»

## Слайд 5 — 6 AI-агентов (~1 мин)

«Почему мы говорим **«AI-агенты»**, а не «чат-бот»?

*Пройти по карточкам агентов.*

| Агент | Что делает | Зачем жюри это важно |
|-------|------------|----------------------|
| **Speech** | Whisper: голос → текст | Реальный канал — сотрудник на ходу |
| **Retriever** | e5 + Qdrant | Не «вся база в prompt», а top-k релевантных фрагментов |
| **Generator** | GigaChat / Ollama | Ответ только по контексту retriever |
| **Evaluator** | Confidence score | Авто vs оператор — **business rule**, не надежда на LLM |
| **Escalation** | Usedesk mock + очередь | L2 не теряет контекст |
| **Analytics** | Redis, weak spots | Замкнутый цикл: **бот → метрики → доработка docs** |

Orchestrator в **FastAPI** — один процесс, агенты как модули. В Kubernetes это один **pod app** + Qdrant + Redis + Ollama — масштабируется без переписывания логики.

**Фраза-якорь:** «Мы не натянули GPT на PDF — мы построили **pipeline с gate на уверенность** и **обратной связью в аналитику**.»»

## Что сказать, чтобы выделиться (шпаргалка спикера)

### 1. Продукт, не демо
«Мы сдали **работающую систему**: бот, web, operator panel, analytics, ingest, тесты брифа — не Figma и не Postman-коллекция.»

### 2. Честность важнее 100%
«Мы **не догнали** 100% — и **специально** показываем кейс эскалации: когда в docs нет ответа, бот **не врёт**. Это production-ready поведение.»

### 3. Dual LLM — зрелость архитектуры
«**GigaChat** на demo — скорость и качество русского на защите. **Ollama** в prod — закрытый контур. Переключение — одна env-переменная, код один.»

### 4. Замкнутый цикл
«Бот → Analytics → weak spots → техписы дополняют Confluence → reindex → бот отвечает лучше. **Flywheel**, не разовый чат.»

### 5. Команда
«**one_commit** — название не случайно: за неделю **end-to-end** от docx до TG-бота с метриками.»

## 1. Краткое описание проекта

### Проблема

Служба поддержки перегружена однотипными вопросами («как настроить фильтр в БО», «кто согласует командировку», «что делать новичку»). Ответы уже есть в объёмной документации, но пользователи не находят разделы, предпочитают «живого» оператора и ждут ответа часами — особенно вне рабочего времени и в пиковые нагрузки.

### Целевая аудитория

- **Пользователи** БАРС-Офис и корпоративных сервисов.
- **Внутренние сотрудники** — новички, менеджеры, офис-менеджеры.



### Решение

**DocHelper Барс** — AI-агент первой линии в **Telegram** и **Web**:

1. Принимает **текст**, **голос** (Whisper), **фото** (Telegram, при наличии vision-модели).
2. **Семантически ищет** фрагменты в docx/pdf/md (Qdrant + multilingual-e5).
3. **Генерирует ответ только из контекста** (GigaChat / Ollama) с **источником** и фрагментом документа.
4. Показывает **иллюстрации из docx** в Web при визуальных вопросах (фильтр, интерфейс, кнопки).
5. При **низкой уверенности** предлагает **оператора**; эскалация с **историей диалога**.
6. **Аналитика**: доля автоответов, топ вопросов, **слабые места документации**, **тренды**; **антиспам** (rate limit).

**Бизнес-цель брифа:** ≥ **40%** тикетов первой линии на автомате.  
**Проверка:** `python scripts/test_brief_cases.py` — 17 вопросов из брифа.  
**Актуальный прогон (27.06.2026):** **16/17 (94%)** · 1 эскалация · 0 ошибок · ~**1,6 сек** ср. — см. [TEST_REPORT.md](TEST_REPORT.md).

## 6. Раскрытие использования AI


| Инструмент                | Задача                                         |
| ------------------------- | ---------------------------------------------- |
| **GigaChat**              | Генерация ответа по найденным фрагментам (RAG) |
| **Ollama**                | Локальная LLM для закрытого контура            |
| **multilingual-e5-base**  | Embeddings для семантического поиска           |
| **faster-whisper**        | Распознавание голосовых (RU)                   |
| **Qdrant**                | Векторный поиск                                |
| **Cursor / AI-ассистент** | Ускорение разработки (код, документация)       |




### AI-конвейер

```
Пользователь → [Whisper] → текст
            → [Retriever: e5 + Qdrant] → фрагменты + иллюстрации docx
            → [LLM: GigaChat/Ollama] → ответ
            → [Confidence gate] → авто / оператор
            → [Escalation + Analytics]
```



### Как проверяли корректность

1. System prompt: ответ **только** из контекста документации.
2. Порог уверенности — ниже порога → оператор, без «уверенной» галлюцинации.
3. Обязательный блок **источника** в UI (документ + фрагмент + скачивание).
4. **17 тест-кейсов** из брифа — **16/17 (94%)**, прогон 27.06.2026, отчёт `TEST_REPORT.md`.
5. Ручная проверка сценариев demo перед защитой.
6. Sanity-check индекса (ненулевые embeddings после ingest).



### Честные ограничения

- GigaChat и Telegram требуют интернет на демо; для прода — Ollama локально.
- Скриншот пользователя в Web — нужен `OLLAMA_VISION_MODEL`.
- Jira / Confluence / MAX — в roadmap, не в MVP.
- Текст ценностей на слайдах docx индексируется через OCR-подготовку при ingest (`docx_sections.py`) — не «заготовки ответов», а попадание картинок в поиск.

## 1. Краткое описание проекта

### Проблема

Служба поддержки Барс Груп перегружена однотипными обращениями: «как сбросить пароль», «как настроить фильтр в БО», «кто согласует командировку», «что делать новичку в первый день». Ответы уже описаны в объёмной документации, но пользователи:

- не находят нужный раздел;
- предпочитают писать «живому» оператору;
- ждут ответа часами, особенно вне рабочего времени и в пиковые нагрузки.

Операторы тратят время на копирование одних и тех же ссылок вместо решения сложных кейсов.

### Целевая аудитория

- **Конечные пользователи** продуктов компании (БАРС-Офис, бизнес-процессы, корпоративные сервисы).
- **Внутренние сотрудники** — новички, менеджеры, офис-менеджеры, которым нужна быстрая справка без отвлечения инженеров.

### Предложенное решение

**DocHelper Барс** — AI-агент первой линии поддержки в **Telegram** и **Web**.

Бот:

1. Принимает вопрос **текстом** или **голосом** (локальный Whisper).
2. Семантически ищет ответ в базе документов (**RAG**: Qdrant + embeddings).
3. Формулирует ответ **только на основе найденных фрагментов** (GigaChat / Ollama) с указанием **источника**.
4. Оценивает **уверенность** — при низкой релевантности предлагает **оператора**.
5. **Эскалирует** диалог в чат поддержки с полной историей.
6. Собирает **аналитику**: доля автоответов, топ вопросов, источники.

**Цель по брифу:** автоматизировать не менее **40%** обращений первой линии.  
**Факт по тест-кейсам заказчика:** **16 из 17** (≈ **94%**) — автоматический ответ с источником.

## 6. Раскрытие использования AI

### Какие AI-инструменты использовались

| Инструмент | Для каких задач |
|------------|-----------------|
| **GigaChat** | Генерация ответа пользователю на основе найденных фрагментов документации (RAG) |
| **Ollama (qwen2.5)** | Локальная LLM — альтернатива для закрытого контура без облака |
| **multilingual-e5-base** | Семантические embeddings для поиска релевантных фрагментов |
| **faster-whisper** | Распознавание голосовых сообщений (русский язык) |
| **Qdrant** | Векторный поиск (не generative, но часть AI-конвейера) |
| **Cursor / AI-ассистент** | Ускорение разработки кода (по необходимости — указать, если использовался) |

### Как устроен AI-конвейер (агенты)

```
Пользователь → [Speech Agent: Whisper] → текст
            → [Retriever: e5 + Qdrant] → фрагменты документов
            → [Generator: GigaChat/Ollama] → ответ + источник
            → [Evaluator: confidence score] → авто / оператор
            → [Escalation Agent] → история в support-чат
            → [Analytics] → метрики и топ вопросов
```

### Как проверяли корректность результатов

1. **RAG-only:** system prompt запрещает отвечать вне контекста документации.
2. **Confidence gate:** при score &lt; 0.45 — не выдаём «уверенный» ответ, предлагаем оператора.
3. **Обязательный источник:** каждый ответ содержит строку «Источник: …».
4. **Тест-кейсы заказчика:** скрипт `scripts/test_brief_cases.py` — 17 вопросов из брифа, результат **94% авто**.
5. **Ручная проверка** команды на demo-сценариях перед защитой.
6. **Sanity-check индекса:** проверка ненулевых embeddings после индексации.
7. **Whisper:** показ распознанного текста пользователю + словарь исправлений доменных терминов («интерфейс бота», «БАРС-Офис»).

### Ограничения и честное раскрытие

- GigaChat требует интернет — для демо; для продакшена предусмотрен **Ollama** локально.
- Whisper на CPU может ошибаться в терминах — для критичных demo-вопросов рекомендуется текст.
- Confluence/Jira интеграции — в roadmap, не в MVP.

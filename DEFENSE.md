## Orchestrator — что это и «pod»

**Orchestrator** — не отдельный ChatGPT, а **координатор pipeline** внутри FastAPI:

1. Принял сообщение (Web/TG)
2. Rate limit → Speech (если голос) → Retriever → Generator → Evaluator
3. Если низкая уверенность → Escalation (тикет + оператор)
4. Параллельно пишет **Analytics** (лог запроса)

**Pod в Kubernetes (если спросят про деплой):**
- **Pod app** — FastAPI + orchestrator (один сервис, внутри — агенты как модули)
- **Pod Qdrant** — векторная база
- **Pod Redis** — сессии, очередь, аналитика, rate limit
- **Pod Ollama** — LLM + vision локально

Смысл: **один вход для пользователя**, внутри — цепочка специализированных агентов, а не один «умный чат».

## Analytics — отдельный агент, не пользовательский бот

**Analytics** — это **отдельный агент/модуль**, не Telegram-бот для сотрудников.

**Что собирает:**
- каждый запрос: вопрос, confidence, авто/эскалация, источник, время
- **weak spots** — темы, где docs не покрывают (низкий conf, нет источника)
- **тrends** — рост тем по дням
- **rate_limited** — спам

**Где смотреть:** `/operator` → вкладка «Аналитика» или `GET /api/analytics`

**Что можно сделать с аналитикой (roadmap / ответ жюри):**
- **Сейчас:** `/operator` → Аналитика, `GET /api/analytics`, Redis `stats:log`, `TEST_REPORT.md`
- **🔜 CSV/Excel** — топ вопросов и weak spots для контент-менеджеров
- **🔜 Confluence/Jira** — repeated weak spots → задачи для техписов
- **🔜 Дашборд** — % авто vs цель 40%, динамика по неделям
- **🔜 Бот-отчёт L2** — утренний digest в TG чат поддержки

**Фраза для защиты:**  
«Пользовательский бот отвечает на вопросы. Analytics-агент **молча** логирует каждый диалог и показывает, **где дыры в документации** — это инструмент для поддержки и техписов, не для конечного пользователя.»

## Быстрые ответы жюри

**«Это ChatGPT?»**  
→ RAG-only. Ответ только из найденных фрагментов + источник. Confidence gate.

**«94% — как считали?»**  
→ 17 вопросов из брифа, скрипт `test_brief_cases.py`, отчёт TEST_REPORT.md, 27.06.2026.

**«Галлюцинации?»**  
→ Кейс №6 (офис-менеджер) — в docs нет → эскалация, не выдумка.

**«Jira / Usedesk?»**  
→ При эскалации создаётся тикет (demo mock USD-…); ответ оператора в панели. В проде — API Usedesk.

**«Git / Confluence reindex?»**  
→ Webhook + кнопка demo в панели оператора. Push в repo → переиндекс Qdrant.

**«Закрытый контур?»**  
→ Ollama + e5 + Qdrant + Whisper на сервере. GigaChat — только для demo.

**«Скриншоты?»**  
→ Vision определяет тип (UI/ошибка/схема) → RAG по распознанному тексту.
